# Guía Rápida: DDL vs DML en SQL

En SQL, el lenguaje se divide en subgrupos según el tipo de tarea que realices. Los dos más importantes son **DDL** (Estructura) y **DML** (Datos).

---

## 1. DDL (Data Definition Language)
El **Lenguaje de Definición de Datos** se utiliza para definir la **estructura** de la base de datos. Es decir, sirve para crear el "esqueleto" o los contenedores donde se guardará la información.

* **Enfoque:** Tablas, Índices, Bases de Datos, Vistas.
* **Impacto:** Los cambios suelen ser permanentes (Auto-commit).

### Comandos Principales
* `CREATE`: Crea un nuevo objeto (ej. una tabla).
* `ALTER`: Modifica la estructura de un objeto existente.
* `DROP`: Elimina un objeto por completo.
* `TRUNCATE`: Borra todos los registros de una tabla, pero mantiene la estructura.

---

## 2. DML (Data Manipulation Language)
El **Lenguaje de Manipulación de Datos** se utiliza para gestionar la **información** que reside dentro de las estructuras creadas por el DDL. Es lo que usamos para el día a día operativo.

* **Enfoque:** Filas, Registros, Datos específicos.
* **Impacto:** Los cambios pueden ser revertidos (`ROLLBACK`) antes de ser confirmados (`COMMIT`).

### Comandos Principales
* `SELECT`: Recupera datos de una o más tablas.
* `INSERT`: Añade nuevos registros a una tabla.
* `UPDATE`: Modifica datos que ya existen.
* `DELETE`: Elimina registros específicos de una tabla.

---

## Comparativa Directa

| Característica | DDL | DML |
| :--- | :--- | :--- |
| **Nivel de operación** | Estructura (Arquitecto) | Datos (Inquilino) |
| **Comandos clave** | `CREATE`, `ALTER`, `DROP` | `SELECT`, `INSERT`, `UPDATE` |
| **Efecto** | Afecta al esquema. | Afecta a las filas. |
| **Persistencia** | Generalmente instantánea. | Permite control de transacciones. |

> **Analogía rápida:** > El **DDL** es quien construye las habitaciones de una casa y decide cuántas ventanas tiene. 
> El **DML** es quien decide qué muebles poner dentro de esas habitaciones y cómo organizarlos.
