# Patrones de Diseño Creacionales en Java

Los patrones creacionales abstraen el proceso de instanciación. Ayudan a que un sistema sea independiente de cómo se crean, componen y representan sus objetos.

---

## Patrón Singleton
**Propósito:** Garantizar que una clase tenga una única instancia y proporcionar un punto de acceso global a ella. Es fundamental para gestionar recursos compartidos como conexiones a bases de datos.

```java
public class DatabaseConnector {
    private static DatabaseConnector instance;
    private String url;

    private DatabaseConnector() {
        this.url = "jdbc:postgresql://localhost:5432/mi_db";
    }

    public static synchronized DatabaseConnector getInstance() {
        if (instance == null) {
            instance = new DatabaseConnector();
        }
        return instance;
    }

    public void connect() {
        System.out.println("Conectado a: " + url);
    }
}
```

## Patrón Factory Method
**Propósito:** Define una interfaz para crear un objeto, pero deja que las subclases decidan qué clase instanciar.

```java
// Interfaz del Producto
interface Transport {
    void deliver();
}

// Productos Concretos
class Truck implements Transport {
    public void deliver() { System.out.println("Entrega por tierra en camión."); }
}

class Ship implements Transport {
    public void deliver() { System.out.println("Entrega por mar en contenedor."); }
}

// Clase Creadora
abstract class Logistics {
    public abstract Transport createTransport();
    
    public void planDelivery() {
        Transport t = createTransport();
        t.deliver();
    }
}

// Creadores Concretos
class RoadLogistics extends Logistics {
    public Transport createTransport() { return new Truck(); }
}

class SeaLogistics extends Logistics {
    public Transport createTransport() { return new Ship(); }
}
```

## Patrón Abstract Factory
**Propósito:** Permite producir familias de objetos relacionados sin especificar sus clases concretas.

```java
// Familia de productos: Muebles
interface Chair { void hasLegs(); }
interface CoffeeTable { void putCoffee(); }

// Implementaciones Concretas
class VictorianChair implements Chair { public void hasLegs() { System.out.println("Silla victoriana."); } }
class VictorianTable implements CoffeeTable { public void putCoffee() { System.out.println("Mesa victoriana."); } }

// Fábrica Abstracta
interface FurnitureFactory {
    Chair createChair();
    CoffeeTable createCoffeeTable();
}

class VictorianFurnitureFactory implements FurnitureFactory {
    public Chair createChair() { return new VictorianChair(); }
    public CoffeeTable createCoffeeTable() { return new VictorianTable(); }
}
```

## Patrón Builder
**Propósito:** Permite construir objetos complejos paso a paso. Útil cuando un objeto tiene muchos parámetros opcionales y requiere una configuración detallada.

```java
public class Pizza {
    private String dough;
    private String sauce;
    private String topping;

    public static class PizzaBuilder {
        private Pizza pizza = new Pizza();

        public PizzaBuilder setDough(String dough) { pizza.dough = dough; return this; }
        public PizzaBuilder setSauce(String sauce) { pizza.sauce = sauce; return this; }
        public PizzaBuilder setTopping(String topping) { pizza.topping = topping; return this; }
        
        public Pizza build() { return pizza; }
    }
}

// Uso:
Pizza miPizza = new Pizza.PizzaBuilder()
                    .setDough("Masa Fina")
                    .setTopping("Pepperoni")
                    .build();
```

## Patrón Prototype
**Propósito:** Permite copiar objetos existentes sin que el código dependa de sus clases. Es eficiente cuando la creación inicial del objeto es costosa en recursos.

```java
interface Prototype {
    Prototype clone();
}

class Document implements Prototype {
    private String content;

    public Document(String content) { this.content = content; }

    @Override
    public Prototype clone() {
        return new Document(this.content);
    }

    @Override
    public String toString() { return "Documento con: " + content; }
}
```
