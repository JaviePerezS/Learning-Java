# 🧠 Principios DRY, WET, KISS, YAGNI

---

## 🟢 DRY — Don't Repeat Yourself

**Don't Repeat Yourself** significa: *No te repitas* o *No te repitas a ti mismo*.

Es un principio que nos enseña a no repetir código o información de manera excesiva.

Por ejemplo, si tenemos una información que debemos mostrar muchas veces en nuestra aplicación, entonces creamos una clase con dicha información y luego la instanciamos o invocamos cada vez que la necesitemos.

### ✅ Ventajas:
- Mantenibilidad del código.
- Mayor legibilidad.
- Reusabilidad.
- Mejor testeabilidad.
- Incremento en la velocidad al desarrollar.

---

## 🔴 WET — Write Everything Twice

**Write Everything Twice** significa: *Repite cada cosa* o *Repite todo*.  
Se pronuncia **"wet"**.

Es considerado un antiprincipio, ya que representa lo contrario a DRY.

Es una mala práctica en la programación porque:
- Genera código duplicado.
- Produce código “sucio”.
- Incrementa la probabilidad de bugs.
- Dificulta el mantenimiento.

---

## 🔵 KISS — Keep It Simple, Stupid

**Keep It Simple, Stupid** significa: *Mantenlo siempre sencillo*.

La implementación debe ser:
- Fácil.
- Clara.
- Directa.

Se deben evitar soluciones excesivamente complejas o sobre diseños cuando no sean necesarios.

### ✅ Ventajas:
- Aumenta la legibilidad del código.
- Incrementa la velocidad de desarrollo.
- Permite obtener código de mayor calidad.
- Reduce la deuda técnica.

---

## 🟡 YAGNI — You Aren't Gonna Need It

**You Aren't Gonna Need It** significa: *No lo vas a necesitar*.

Este principio nos indica que no debemos escribir código que no sea necesario.  
Debemos enfocarnos únicamente en lo que se nos pidió implementar.

Si no se aplica:

- Incrementa el tiempo de desarrollo.
- Aumenta la complejidad para testear.
- Reduce la legibilidad.
- Incrementa costos sin generar beneficios reales.

---

# 📊 Notación Asintótica

La **notación asintótica** se utiliza para analizar el coste de un algoritmo.

Permite medir:
- Tiempo de ejecución.
- Uso de memoria.
- Comportamiento del algoritmo a medida que crecen los datos.
