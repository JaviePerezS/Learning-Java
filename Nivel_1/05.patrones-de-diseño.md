# Patrones de Diseño

## 1. Patrones Creacionales
*Controlan el proceso de instanciación para desacoplar el sistema de la creación de objetos.*

### Factory Method
* **Definición:** Clase basada en polimorfismo que centraliza la creación de objetos, permitiendo que las subclases decidan qué instancia específica retornar.
* **Ejemplo:**
```java
// El método fábrica permite cambiar el tipo de transporte sin tocar el cliente
public interface Transporte { void entregar(); }

public class TransporteFactory {
    public Transporte getTransporte(String tipo) {
        if (tipo.equals("MARITIMO")) return new Barco();
        return new Camion();
    }
}
```

### Abstract Factory
* **Definición:** Provee una interfaz para crear **familias** de objetos relacionados (ej. componentes de una UI) sin especificar sus clases concretas.
* **Ejemplo:**
```java
// Crea una familia de componentes compatibles entre sí (Botón y Checkbox)
public interface GUIFactory {
    Boton crearBoton();
    Checkbox crearCheckbox();
}

public class WinFactory implements GUIFactory { ... }
public class MacFactory implements GUIFactory { ... }
```

### Singleton
* **Definición:** Garantiza una única instancia de una clase concreta.
* **Desventajas:** Dificulta el testing al introducir estado global (requiere mocks complejos).
* **Ejemplo:**
```java
public class DatabaseConfig {
    private static DatabaseConfig instance;
    private DatabaseConfig() { } // Constructor privado
    
    public static DatabaseConfig getInstance() {
        if (instance == null) instance = new DatabaseConfig();
        return instance;
    }
}
```

## 2. Patrones Estructurales
*Organizan clases y objetos para formar estructuras mayores y flexibles.*

### Decorator
* **Definición:** Componente que envuelve a otro componente, creando capas de funcionalidad adicional sin alterar la clase original.
* **Ejemplo:**
```java
// Capas de funcionalidad: Notificador -> SMSDecorator -> WhatsAppDecorator
public abstract class NotificadorDecorator implements Notificador {
    protected Notificador envuelto;
    public void enviar(String msg) { envuelto.enviar(msg); }
}
```

## 3. Patrones de Comportamiento
*Gestionan algoritmos y la comunicación entre los objetos.*

### Strategy
* **Definición:** Encapsula diferentes lógicas o algoritmos en clases independientes para intercambiarlos fácilmente en tiempo de ejecución.
* **Ejemplo:**
```java
// Permite cambiar el método de pago sin modificar la clase Carrito
public interface EstrategiaPago { void pagar(int monto); }

public class CarritoCompra {
    private EstrategiaPago estrategia;
    public void ejecutarPago(int total) { estrategia.pagar(total); }
}
```
