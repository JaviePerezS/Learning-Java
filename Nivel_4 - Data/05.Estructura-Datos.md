# 🚀 Las 8 Estructuras de Datos

### 1. 📦 Array (Arreglo)
Es una lista de elementos guardados uno tras otro en la memoria, como una fila de cajas numeradas.
* **Ejemplos:**
    * Píxeles de una foto.
    * Inventarios de objetos en un juego.
    * Lista de puntajes de una partida.
* **Ventajas:** Lectura instantánea por índice.
* **Desventajas:** Lento para insertar o borrar en medio.

### 2. 🔗 Linked List (Lista Enlazada)
Estructura de nodos esparcidos donde cada uno contiene su dato y la dirección del siguiente, como eslabones de una cadena.
* **Ejemplos:**
    * Feed de Instagram.
    * Listas de reproducción de música.
    * Historial "atrás/adelante" del navegador.
* **Ventajas:** Inserción y eliminación muy rápidas sin mover otros datos.
* **Desventajas:** No tiene acceso directo; hay que recorrerla nodo por nodo.

### 3. 📚 Stack (Pila)
Gestión de datos bajo el principio LIFO (Último en entrar, primero en salir), como una pila de platos.
* **Ejemplos:**
    * El botón "Deshacer" (Ctrl+Z).
    * El botón "Atrás" de las pestañas.
    * Gestión de funciones en programación.
* **Ventajas:** Control perfecto de procesos reversibles.
* **Desventajas:** Acceso restringido; debes vaciar la cima para llegar al fondo.

### 4. 👥 Queue (Cola)
Estructura basada en el principio FIFO (Primero en llegar, primero en salir), como la fila del supermercado.
* **Ejemplos:**
    * Cola de impresión de documentos.
    * Mensajes de WhatsApp esperando señal.
    * Jugadores esperando entrar a una partida (Matchmaking).
* **Ventajas:** Garantiza un orden justo de procesamiento.
* **Desventajas:** El primer elemento puede bloquear el avance de los demás.

### 5. 🔑 Hash Table (Tabla Hash)
Asocia una "clave" (nombre) a un valor mediante una función para encontrar datos de forma inmediata.
* **Ejemplos:**
    * Contactos del móvil.
    * Sistema de login (Correo -> Cuenta).
    * Diccionarios de idiomas.
* **Ventajas:** Búsqueda ultra rápida entre millones de registros.
* **Desventajas:** Alto consumo de memoria.

### 6. 🌳 Árbol Binario de Búsqueda
Organiza datos dividiendo el camino: menores a la izquierda y mayores a la derecha. Es como el juego de adivinar un número descartando mitades.
* **Ejemplos:**
    * Carpetas y subcarpetas del sistema.
    * Árbol genealógico.
    * Comentarios y sus respuestas en foros.
* **Ventajas:** Búsquedas muy veloces al descartar el 50% de los datos en cada paso.
* **Desventajas:** Si no se mantiene equilibrado, pierde toda su velocidad.

### 7. 🏥 Heap (Montículo de Prioridad)
Estructura viva que siempre mantiene al "más urgente" en la cima. Si el líder se va, otro sube de inmediato para ocupar su lugar de prioridad.
* **Ejemplos:**
    * Triage de urgencias en un hospital.
    * Prioridad de procesos en el procesador (CPU).
    * Rankings de jugadores en tiempo real.
* **Ventajas:** Gestión de prioridades inmediata sin ordenar toda la lista.
* **Desventajas:** Ineficiente para búsquedas generales; solo sirve para el primero.

### 8. 🕸️ Grafo
Red de puntos (nodos) y conexiones (hilos) que representa relaciones. Es la estructura que mejor imita al mundo real y sus conexiones.
* **Ejemplos:**
    * Redes Sociales (quién conoce a quién).
    * Mapas y GPS (ciudades y carreteras con tráfico).
    * Recomendaciones de Netflix/Amazon (películas relacionadas).
    * Internet (páginas conectadas por links).
* **Ventajas:** Modela perfectamente conexiones complejas y permite encontrar rutas óptimas.
* **Desventajas:** Muy pesado de procesar cuando hay millones de conexiones.
