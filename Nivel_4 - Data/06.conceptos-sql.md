# 📊 Conceptos de Consultas y Agrupación SQL

## I. Sentencias de Agrupación en Detalle

### **GROUP BY**
Representa la segmentación de los datos. Esta cláusula colapsa múltiples filas en una sola fila por cada grupo definido, permitiendo organizar la información bajo un criterio común.

**Consulta:**
```sql
SELECT departamento, COUNT(id_empleado) AS total_empleados
FROM empleados
GROUP BY departamento;
```

### **Funciones de Agregado**
Representan las operaciones matemáticas que dan sentido a la agrupación. Sin ellas, el `GROUP BY` solo actuaría como un eliminador de duplicados, perdiendo su capacidad analítica.

**Consulta:**
```sql
SELECT 
    categoria_producto,
    SUM(precio) AS ingresos_totales,
    AVG(precio) AS precio_promedio,
    MAX(precio) AS producto_mas_caro
FROM ventas
GROUP BY categoria_producto;
```

### **HAVING**
Representa la validación de los resultados calculados. A diferencia del `WHERE`, esta cláusula tiene la capacidad de evaluar el resultado de una función de agregado (como una suma o un conteo).
**Consulta:**
```sql
SELECT sucursal, SUM(monto_venta) AS venta_total
FROM transacciones
GROUP BY sucursal
HAVING SUM(monto_venta) > 50000; -- Filtra grupos resultantes, no filas base.
```

## II. 🧠 Conceptos Avanzados de Consultas

### **1. El Orden de Ejecución (Lógica Interna)**
Representa la ruta que sigue el motor de la base de datos para procesar la información. Entender este flujo es crucial para evitar errores de referencia de columnas y optimizar el rendimiento.

* **Orden Real:** `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY`.
* **Dato Clave:** No es posible utilizar un **Alias** definido en el `SELECT` dentro del `WHERE`, debido a que el motor filtra los registros antes de procesar las columnas finales.

### **2. CTE (Common Table Expressions) - Cláusula `WITH`**
Representa la creación de tablas temporales con nombre para modularizar consultas extensas. Su uso es altamente recomendado sobre las subconsultas debido a su legibilidad y facilidad de mantenimiento.
**Consulta:**
```sql
WITH ventas_por_cliente AS (
    SELECT id_cliente, SUM(total) AS gasto_total
    FROM facturas
    GROUP BY id_cliente
)
SELECT id_cliente 
FROM ventas_por_cliente 
WHERE gasto_total > 1000;
```

### **3. DISTINCT**
Representa el filtro de unicidad a nivel de fila. Se utiliza cuando el objetivo es identificar los valores existentes en un conjunto de datos sin considerar las repeticiones.
**Consulta:**
```sql
SELECT DISTINCT pais_residencia 
FROM usuarios;
```

### **4. CASE WHEN (Lógica Condicional)**
Representa la capacidad de implementar lógica de decisión (tipo IF/ELSE) directamente en la consulta. Permite categorizar datos o transformar valores dinámicamente.
**Consulta:**
```sql
SELECT nombre,
       CASE 
           WHEN salario > 3000 THEN 'Alto'
           ELSE 'Estándar'
       END AS rango_salarial
FROM empleados;
```
