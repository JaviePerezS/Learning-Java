# 📉 Lógica de Filtrado y Transformación de Datos

### 🔍 WHERE
Representa el filtro de filas individuales aplicado antes de la agrupación. Actúa directamente sobre los datos base de las tablas.

### 📊 HAVING
Representa el filtro de grupos aplicado después de la agregación. Se utiliza para condicionar resultados de funciones como SUM, AVG o COUNT.

### 🏆 QUALIFY
Representa el filtro para Funciones de Ventana (Window Functions). Permite filtrar resultados de funciones como RANK() o ROW_NUMBER() sin usar subconsultas.

### 💧 COALESCE
Representa la función de limpieza que selecciona el primer valor no nulo entre varias columnas. Es vital para el tratamiento de nulos y la consolidación de campos.

| Operación | Se aplica sobre... | Propósito Técnico |
| :--- | :--- | :--- |
| **WHERE** | Filas Base | Filtrado inicial de registros. |
| **HAVING** | Grupos | Filtrado de totales y cálculos. |
| **QUALIFY**| Ventanas | Filtrado de rankings y particiones. |
| **COALESCE**| Valores | Manejo de nulos y valores por defecto. |
