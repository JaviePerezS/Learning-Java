# Diccionario Técnico de Joins en SQL

Un **JOIN** es una operación fundamental en SQL que permite combinar filas de dos o más tablas basándose en una columna relacionada entre ellas (generalmente llaves primarias y foráneas).

---

### 📊 Tablas de Origen

**tabla: usuarios**

| :--- | :--- |
| 1 | ana |
| 2 | beto |
| 3 | carla |

**tabla: pedidos**
| id_pedido | id_usuario | producto |
| :--- | :--- | :--- |
| 10 | 1 | teclado |

| 12 | 4 | monitor |

### 🛠️ Tipos de JOIN y ejemplos de código

#### 🎯 INNER JOIN
Devuelve únicamente los registros que tienen una coincidencia en ambas tablas.



**Código:**
```sql
select usuarios.nombre, pedidos.producto
from usuarios
inner join pedidos on usuarios.id = pedidos.id_usuario;
```

**Descripción:** Solo verás a **ana** con su teclado y mouse. Beto y Carla no aparecen porque no tienen pedidos, y el monitor no aparece porque su usuario (4) no existe en la tabla de usuarios.

**Salida:**
| nombre | producto |
| :--- | :--- |
| ana | teclado |
| ana | mouse |

---

#### ⬅️ LEFT JOIN
Mantiene todos los registros de la tabla izquierda (`usuarios`) y añade los datos de la derecha donde haya coincidencia. Si no hay, rellena con **NULL**.



**Código:**
```sql
select usuarios.nombre, pedidos.producto
from usuarios
left join pedidos on usuarios.id = pedidos.id_usuario;
```

**Descripción:** Verás a **ana**, **beto** y **carla**. Como Beto y Carla no tienen pedidos registrados, en la columna producto aparecerá **NULL**.

**Salida:**
| nombre | producto |
| :--- | :--- |
| ana | teclado |
| ana | mouse |
| beto | **NULL** |
| carla | **NULL** |

---

#### ➡️ RIGHT JOIN
Mantiene todos los registros de la tabla derecha (`pedidos`) y añade los de la izquierda donde coincidan.



**Código:**
```sql
select usuarios.nombre, pedidos.producto
from usuarios
right join pedidos on usuarios.id = pedidos.id_usuario;
```

**Descripción:** Verás los pedidos de **ana** (teclado y mouse) y el **monitor**. Como el monitor pertenece al usuario 4 (que no existe en nuestra tabla), el nombre del usuario saldrá como **NULL**.

**Salida:**
| nombre | producto |
| :--- | :--- |
| ana | teclado |
| ana | mouse |
| **NULL** | monitor |

---

#### 🔄 FULL OUTER JOIN
Muestra todos los registros de ambas tablas. Si no hay coincidencia en algún lado, rellena con **NULL**.



[Image of SQL Full Join Venn diagram]


**Código:**
```sql
select usuarios.nombre, pedidos.producto
from usuarios
full outer join pedidos on usuarios.id = pedidos.id_usuario;
```

**Descripción:** Verás a todos los usuarios y todos los pedidos. Aparecerán **NULLs** tanto para los usuarios que no han comprado (Beto y Carla) como para los pedidos que no tienen un usuario válido (Monitor).

**Salida:**
| nombre | producto |
| :--- | :--- |
| ana | teclado |
| ana | mouse |
| beto | **NULL** |
| carla | **NULL** |
| **NULL** | monitor |

---

#### ✖️ CROSS JOIN
Combina cada registro de la tabla A con todos los registros de la tabla B (producto cartesiano).

**Código:**
```sql
select usuarios.nombre, pedidos.producto
from usuarios
cross join pedidos;
```

**Descripción:** Si hay 3 usuarios y 3 pedidos, obtendrás **9 filas**. Cada usuario se repite con cada uno de los productos de la tabla de pedidos, sin importar la relación de ID.

**Salida:**
| nombre | producto |
| :--- | :--- |
| ana | teclado |
| ana | mouse |
| ana | monitor |
| beto | teclado |
| beto | mouse |
| beto | monitor |
| carla | teclado |
| carla | mouse |
| carla | monitor |

---

### 📝 Diccionario Técnico de Joins

| Tipo | Qué representa | Manejo de No Coincidencias |
| :--- | :--- | :--- |
| **Inner** | Intersección | Se eliminan del resultado. |
| **Left** | Prioridad Izquierda | Se mantienen (Derecha = **NULL**). |
| **Right** | Prioridad Derecha | Se mantienen (Izquierda = **NULL**). |
| **Full** | Unión Total | Se mantienen todos (con **NULLs**). |
| **Cross** | Combinatoria | Multiplica todas las filas entre sí. |
