# 🛡️ El Principio ACID
En el mundo de las bases de datos, **ACID** es un conjunto de cuatro propiedades que garantizan que las transacciones se realicen de forma segura. Piensa en esto como el "contrato de confianza" entre el sistema y tus datos.

---

### 1. **A**tomicity (Atomicidad)
> **"Todo o nada"**

Una transacción se trata como una unidad "atómica" (indivisible). Si la transacción tiene 5 pasos y el paso 4 falla, el sistema deshace los 3 anteriores y deja todo como estaba al principio.

* **Ejemplo:** Si transfieres dinero, primero se resta de tu cuenta y luego se suma en la otra. Si el sistema falla antes de sumar, la operación se cancela y recuperas tu dinero en tu cuenta original.

### 2. **C**onsistency (Consistencia)
> **"Reglas sagradas"**

Asegura que la base de datos pase de un estado válido a otro estado válido, respetando todas las reglas (restricciones, cascadas, disparadores). No se permite ninguna operación que rompa la integridad de los datos.

* **Ejemplo:** Si una regla dice que el inventario no puede ser menor a cero, la base de datos rechazará cualquier venta que deje el stock en números negativos.

### 3. **I**solation (Aislamiento)
> **"Cada uno en su turno"**

Permite que múltiples transacciones ocurran al mismo tiempo sin que una interfiera con la otra. Para el sistema, es como si las transacciones se ejecutaran una tras otra, de forma aislada.

* **Ejemplo:** Si dos personas intentan comprar el último pastel en una tienda online al mismo tiempo, el aislamiento asegura que solo una lo logre y la otra vea "Agotado", evitando que el sistema venda el mismo pastel dos veces.

### 4. **D**urability (Durabilidad)
> **"A prueba de fallos"**

Una vez que el sistema confirma que una transacción fue exitosa, los cambios son permanentes. Incluso si hay un corte de energía o el servidor se apaga de repente, los datos no se perderán.

* **Ejemplo:** En cuanto recibes el mensaje de "Pedido confirmado", esa información ya está grabada en el disco duro y sobrevivirá a cualquier reinicio del sistema.

---

### 📊 Resumen de Control

| Letra | Propiedad | Concepto Clave |
| :--- | :--- | :--- |
| **A** | **Atomicidad** | Si algo falla, se revierte todo. |
| **C** | **Consistencia** | Se respetan las reglas del negocio. |
| **I** | **Aislamiento** | Las operaciones no se mezclan. |
| **D** | **Durabilidad** | Lo guardado es permanente. |
